ACCESSIBILITY: 

1. Navigation (<nav>)

General principles:

Should have aria-label if multiple navs exist (e.g., aria-label="Main navigation")

All focusable elements inside must be reachable via Tab

Dropdown buttons:

aria-haspopup="true" → indicates menu exists

aria-expanded="false" → will toggle when open

Links should clearly describe their destination

Keyboard behavior:

Tab moves through top-level links/buttons

Arrow keys can optionally navigate dropdown items (advanced)

2. Buttons

For interactive controls like dropdowns or toggles:

Default <button> is keyboard focusable (Enter/Space triggers click)

If it expands/collapses something:

aria-expanded="false" → toggled to "true" when open

aria-controls="id-of-menu" → points to the element being controlled

Don’t rely on mouse-only events; keyboard must activate them

3. Links (<a>)

Must have meaningful text (avoid “click here”)

Navigate via Enter key by default

If a link opens a new page/tab, optionally add a visual or screen reader cue

4. Forms / Input elements (for future sections)

<label> should be linked to input (for attribute)

All inputs must be keyboard operable

Use aria-required or aria-invalid for validation states

Tab order should flow logically

5. Sections / Headings

Use headings <h1>-<h6> to create document outline

Ensure Tab order follows logical reading order

Skip links (e.g., <a href="#main" class="skip-link">Skip to main content</a>) help keyboard users jump over nav

6. Modals / Popups (for future dropdowns or overlays)

Trap focus inside modal while open

Esc closes modal

Return focus to triggering button/link after close

aria-modal="true" and role="dialog"

7. Keyboard accessibility rules in general

Everything interactive must be reachable by Tab

Buttons: activated with Enter/Space

Links: activated with Enter

Avoid keyboard traps

Logical focus order matching reading order

Visual focus indicators (outline) — can add later in CSS

✅ Summary

Right now, your mental checklist for header navigation:

Logo → focusable link

Top nav items → buttons (dropdown) or link (pricing)

Right-side actions → focusable links

Dropdown buttons → ARIA attributes for expansion

Keyboard order → logo → nav items → actions
